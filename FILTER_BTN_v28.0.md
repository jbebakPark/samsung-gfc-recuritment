## 🐛 문제점 (v27.0)

### 스크린샷 분석
사용자 제공 스크린샷에서 발견된 문제:
1. **필터 버튼 텍스트가 거의 안 보임**
   - "포털/웹", "유튜브", "블로그" 텍스트가 흰색으로만 보임
   - 파란색 배경과 대비가 약함
   
2. **active 상태 버튼의 가독성 문제**
   - active 버튼: 파란색 배경 + 흰색 텍스트
   - 색상 대비는 있지만 폰트가 너무 얇음 (font-weight: 600)
   
3. **일반 상태 버튼의 문제**
   - 흰색 배경 + 파란색 텍스트
   - 그림자가 약해서 버튼처럼 안 보임

## 🚀 주요 개선점

### 1. 폰트 굵기 강화
- **Before**: `font-weight: 600`
- **After**: `font-weight: 700`
- **효과**: 텍스트가 더 선명하고 읽기 쉬움

### 2. 그림자 효과 추가
- **일반 상태**: `box-shadow: 0 2px 8px rgba(3, 78, 162, 0.2)`
- **active 상태**: `box-shadow: 0 4px 16px rgba(3, 78, 162, 0.4)`
- **효과**: 버튼이 "눌러질 수 있는" 것처럼 보임, 입체감 증가

### 3. 호버 애니메이션
- **Before**: 단순 색상 변경
- **After**: 색상 변경 + 위로 2px 이동 + 그림자 강화
- **효과**: 인터랙티브한 느낌, 사용자 피드백 명확

### 4. 인라인 스타일 적용
- **이유**: CSS 캐시 우회
- **장점**: 즉시 적용, 브라우저 캐시 무시
- **단점**: HTML 파일이 약간 커짐 (허용 가능)

## 🎓 교훈

### 1. 인라인 스타일의 강력함
- 외부 CSS는 캐시 문제로 적용 안 될 수 있음
- 인라인 스타일은 **항상 우선 적용**됨
- `!important` + 인라인 = **최강의 조합**

### 2. 폰트 굵기의 중요성
- `font-weight: 600` vs `700`의 차이가 생각보다 큼
- 특히 작은 화면에서는 **굵은 폰트**가 필수

### 3. 그림자 효과
- 버튼은 **클릭 가능해 보여야** 함
- `box-shadow`는 단순 장식이 아니라 **UX 개선 도구**

### 4. 호버 애니메이션
- `transform: translateY(-2px)` - 작은 변화가 큰 차이
- 사용자에게 **즉각적인 피드백** 제공
