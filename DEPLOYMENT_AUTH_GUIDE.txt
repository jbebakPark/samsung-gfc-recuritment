AUTHENTICATION REQUIRED:
--------------------------------------------------------------------------------

This is a sandbox environment. To deploy, you need a Firebase CI token.

OPTION 1: Get Firebase CI Token (RECOMMENDED)
--------------------------------------------------------------------------------

Run this command on YOUR LOCAL MACHINE (not in sandbox):

   firebase login:ci

This will:
1. Open a browser for Google authentication
2. Generate a CI token (looks like: 1//0abc123...)
3. Display the token in terminal

Then, use the token to deploy:

   FIREBASE_TOKEN="your-token-here" npx firebase deploy --only hosting

OPTION 2: Use GitHub Actions (AUTOMATED)
--------------------------------------------------------------------------------

The repository already has GitHub Actions workflow configured.
Simply push to main branch and it will auto-deploy:

   git push origin main

GitHub Actions will handle authentication automatically.

OPTION 3: Deploy from Local Machine
--------------------------------------------------------------------------------

1. Clone the repository to your local machine:
   git clone https://github.com/jbebakPark/samsung-gfc-recuritment.git
   cd samsung-gfc-recuritment

2. Install Firebase CLI:
   npm install -g firebase-tools

3. Login to Firebase:
   firebase login

4. Deploy:
   firebase deploy --only hosting

GITHUB ACTIONS AUTO-DEPLOY:
--------------------------------------------------------------------------------

Repository: https://github.com/jbebakPark/samsung-gfc-recuritment
Actions: https://github.com/jbebakPark/samsung-gfc-recuritment/actions

To enable auto-deploy on every git push:

1. Get Firebase CI token on local machine: firebase login:ci
2. Copy the token
3. Go to: https://github.com/jbebakPark/samsung-gfc-recuritment/settings/secrets/actions
4. Click "New repository secret"
5. Name: FIREBASE_TOKEN
6. Value: Paste your token
7. Click "Add secret"

After this, every git push to main will auto-deploy!
